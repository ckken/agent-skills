# 提示词栈审计量表

对每个维度按 0–3 分评分：0=缺失或高风险，1=明显不足，2=基本可用，3=清晰且有验证。分数只用于定位，不替代行为评测。

| 维度 | 重点检查 | 3 分标准 |
|---|---|---|
| 结果合同 | 目标、成功标准、完成定义 | 结果可观察，完成前条件明确 |
| 来源与优先级 | 规则来源、范围、所有者、覆盖关系 | 无隐式覆盖，嵌套边界清晰 |
| 重复与冲突 | 字面重复、语义重复、相反要求 | 单一事实来源，无重要矛盾 |
| 权限与自主性 | 读、改、外部写入、破坏性操作 | 授权矩阵集中且不重复 |
| 工具设计 | 使用条件、输入、返回、错误、回退 | 只暴露相关工具，错误路径明确 |
| 检索与证据 | 必要事实、引用、缺失证据处理 | 重要结论可追溯，推断单独标记 |
| 停止与回退 | 重试、降级、追问、结束 | 能停止无效循环并保留正确性 |
| 输出控制 | 结构、详细度、人格、协作方式 | 保留内容优先级明确，无模糊风格词 |
| 上下文与状态 | 缓存前缀、压缩、记忆、陈旧状态 | 稳定与动态内容分离，过期可失效 |
| 多 Agent | 委派价值、并发、共享写状态、整合 | 独立任务并行，单写者，主 Agent 验收 |
| 框架生命周期 | 安装、加载、作用域、替代、退役和残留 | 目标状态明确，运行时与文件状态分别验证 |
| 安全边界 | 提示注入、秘密、私有数据、副作用 | 不可信内容只作数据，敏感信息不回显 |
| 评测与可观测性 | 版本、任务集、trace、指标、回滚 | 改动可归因、可复现、可回滚 |

## 硬门槛

出现以下任一情况，不应仅凭 Token 下降判定通过：

- 安全、权限或范围违规增加；
- 必要事实、引用或验证被删除；
- 用户明确值被通用默认覆盖；
- 外部写入或破坏性动作失去确认边界；
- 将按需化或停用擅自扩大为用户级卸载、深度清理或跨宿主删除；
- 评测同时改变多个不可归因变量；
- 只比较最终文案，不检查工具动作和中间副作用。

## 风险级别

- **P0**：可能泄密、越权、执行破坏性动作或覆盖高优先级指令。
- **P1**：可能产生错误结果、缺少关键证据、范围扩张或验证失效。
- **P2**：导致无效追问、工具循环、延迟、Token 或维护成本上升。
- **P3**：措辞、组织和可读性问题，不直接改变核心行为。

先处理 P0/P1，再处理 P2；不要为 P3 美化破坏稳定行为。
