# IP 封面设计

围绕“一个主题、一条信息、一个稳定 IP”设计封面。IP 是品牌识别和视觉引导者，不应遮挡主题，也不应让封面退化为角色设定图。

## 开始前

先查看 IP 参考图和目标载体。若尚未建立身份 brief，先执行 `$ip-visual-identity` 的身份检查逻辑。

确认：

- 封面用于 README、文章、网页 Hero 还是社媒；
- 精确尺寸或宽高比；
- 标题、副标题和品牌署名；
- 需要突出的是合集、产品、功能还是活动；
- 当前仓库或页面已有的视觉风格。

常见尺寸见 [cover-specs.md](references/cover-specs.md)。

## 设计原则

- 标题是第一视觉层级，IP 是第二层级，辅助模块是第三层级。
- 合集封面要呈现多个能力模块，不让某一个功能独占画面。
- 保留大面积呼吸空间，避免把所有信息塞进卡片。
- 文字只保留标题、短副标题和必要署名。
- 重要内容放在安全区内，适配 GitHub 和移动端缩放。
- 不把 IP 参考图中的设定说明、视图标签或背景边框复制进封面。

## 执行流程

### 1. 读取上下文

读取目标 README、页面标题、产品说明或用户文案。若替换现有封面，同时确认当前图片路径和页面首屏结构。

### 2. 选择构图

优先选择一种：

- **角色策展人**：IP 位于一侧，组织多个能力模块，适合合集与工具库；
- **角色使用者**：IP 正在操作产品界面，适合应用和功能；
- **角色引导者**：IP 指向标题或核心成果，适合文章和发布页；
- **品牌场景**：IP 融入统一世界观，适合活动和频道 Banner。

### 3. 编写生成提示词

提示词必须包含：

- IP 身份锚点和原始参考图；
- 目标尺寸、构图与安全区；
- 标题的精确文本；
- 主题层级和禁止单一功能喧宾夺主的要求；
- 材质、光线、背景和品牌色；
- “不生成额外文字、乱码、第三方 Logo”的约束。

标题较长或必须像素级准确时，优先生成无字底图，再用可靠排版工具添加文字。

### 4. 生成与视觉验收

生成后检查：

- IP 是否与参考图一致；
- 标题是否精确、可读、未被裁切；
- 缩小到 README 显示宽度后是否仍清晰；
- 是否明确表达目标主题；
- 是否存在多余文字、错误 Logo、肢体或连接结构异常。

不合格时优先重新生成，不直接用低质量结果替换现有封面。

### 5. 压缩并接入

使用 `$agent-image-optimizer`：

```bash
agent-image-opt --json inspect <cover>
agent-image-opt --json optimize <cover> --preset illustration --max-width 1600 --dry-run
agent-image-opt --json optimize <cover> --preset illustration --max-width 1600
```

README 封面优先交付 WebP，并保留合理清晰度。修改引用后验证文件存在、尺寸正确且 Git diff 只包含目标资源。

## 输出合同

最终返回：

1. 封面文件和尺寸；
2. 使用的标题与构图；
3. IP 一致性检查结果；
4. 压缩前后体积；
5. 已修改的 README 或页面引用；
6. 未完成的人工选择或风险。
