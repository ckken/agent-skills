# IP 宣传海报

宣传图必须先让受众看懂“宣传什么”，再感受到 IP。每张图只承担一个核心传播目标，不把功能列表直接堆成说明书。

## 开始前

先查看 IP 参考图。若没有稳定身份 brief，先执行 `$ip-visual-identity` 的身份检查逻辑。

从用户材料中提取：

- 核心卖点；
- 目标受众；
- 发布场景和平台；
- 主标题、行动语和必要日期；
- 品牌色、禁用元素和已有视觉资产。

用户没有给完整文案时，可压缩为一条主标题和一条短副标题，但不要杜撰价格、数据、日期或承诺。

规格建议见 [promo-specs.md](references/promo-specs.md)。

## 视觉叙事

从以下结构中选择一种：

- **能力展示**：IP 正在使用或展示核心功能；
- **结果展示**：IP 与最终成果同框，强调收益；
- **发布宣告**：大标题、产品主体和 IP 共同形成发布感；
- **场景故事**：用一个具体使用场景表现问题与解决方案；
- **系列卡片**：一个主视觉加多个统一规格的功能子图。

一个主视觉只选一种结构。

## 执行流程

### 1. 建立信息层级

按以下顺序组织：

1. 主标题；
2. 产品、功能或活动主体；
3. IP 角色；
4. 一句副标题或行动语；
5. 必要的品牌署名。

不使用超过三层的小字说明。二维码、日期和价格必须来自用户提供的可靠素材。

### 2. 编写生成提示词

提示词包含：

- 原始 IP 参考图和身份锚点；
- 宣传目标与受众；
- 精确画幅、主体位置和留白；
- 产品或成果如何与 IP 互动；
- 主标题的精确文字；
- 品牌材质、灯光、色彩和情绪；
- 禁止乱码、额外标签、第三方 Logo 和角色漂移。

含较多中文、二维码、价格或日期时，优先生成无字主视觉，再单独排版，避免让生成模型承担精确排字。

### 3. 生成主视觉

先生成一个母版，再派生不同尺寸。不要为每个平台完全重新生成角色，以免系列物料出现身份漂移。

派生时保持：

- 同一 IP 身份和服装；
- 同一主色与光线；
- 同一标题语义；
- 同一产品主体；
- 相近的角色姿势和镜头方向。

### 4. 验收

检查：

- 三秒内能否理解卖点；
- IP 是否仍是参考图中的同一角色；
- 产品主体是否真实、完整且没有错误界面；
- 文案是否逐字准确；
- 是否出现虚假数据、错误徽标或无来源承诺；
- 不同尺寸是否只是合理重排，而非变成不同 Campaign。

### 5. 压缩交付

使用 `$agent-image-optimizer` 先预览再写入：

```bash
agent-image-opt --json inspect <poster-or-directory> --recursive
agent-image-opt --json optimize <poster-or-directory> --recursive --preset poster --dry-run
agent-image-opt --json optimize <poster-or-directory> --recursive --preset poster --report image-opt-report.json
```

保留主视觉源文件或高质量母版；发布文件优先使用 WebP。含细小文字时可改用 `ui` 预设并进行视觉复核。

## 输出合同

最终返回：

1. 主视觉和各平台尺寸；
2. 使用的核心卖点与文案；
3. IP 一致性和文字准确性检查；
4. 压缩结果与 JSON 回执位置；
5. 哪个文件是母版、哪个是发布版；
6. 未验证的产品界面、数据或外部素材。
