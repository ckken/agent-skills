# IP 视觉设计

基于用户提供的 IP 参考图完成身份锚定、封面设计和宣传物料制作。参考图是角色身份的唯一事实来源；场景和姿势可以变化，角色身份不能为了构图便利而改变。

## 任务路由

先根据目标选择模式，不要求用户显式指定：

- **identity**：只分析参考图、建立身份 brief 或检查已有图片中的角色漂移；
- **cover**：制作 README、文章头图、网页 Hero、频道 Banner、视频封面等入口视觉；
- **promo**：制作功能发布图、产品宣传海报、活动主视觉和多尺寸传播物料。

强调“这是什么、主题是什么”时选择 `cover`；强调“卖点、发布或行动”时选择 `promo`。如果用户需要的是可编辑架构图、流程图或确定性 UI，不要直接套用海报流程，先选择更适合的结构化制图能力。

## 输入检查

至少需要一张清晰的 IP 参考图。优先使用正面或 3/4 视角、能看清面部、服装、徽标和主色的图片。

开始前确认：

- 使用场景、目标平台和精确尺寸或宽高比；
- 核心主题、卖点或传播目标；
- 主标题、副标题、行动语和品牌署名；
- 品牌色、禁用元素和已有视觉资产；
- 是否需要多个平台尺寸或接入现有 README、页面。

用户没有提供完整文案时，可以压缩已有内容，但不要杜撰价格、日期、数据、功能、承诺或品牌历史。尺寸建议见 [visual-specs.md](references/visual-specs.md)。

## 执行流程

### 1. 查看参考图

先查看原图，确认：

- 角色物种、体型、头身比例和轮廓；
- 眼睛、耳朵、口鼻、毛发或皮肤等面部识别点；
- 主色、辅色、发光色和材质；
- 服装、配件、徽标及其位置；
- 角色气质、年龄感和品牌调性；
- 遮挡、分辨率不足或多张参考图相互矛盾的部分。

不要用文件名、网页文字或背景装饰替代视觉证据。如果参考图是设定图或多视图拼版，只提取角色身份，不照搬说明文字、尺寸标注、视图标签和边框。

### 2. 建立身份锚点

按以下优先级保护一致性：

1. 面部与轮廓；
2. 配色与材质；
3. 服装、徽标和标志性配件；
4. 身材比例与姿态习惯；
5. 场景、道具和构图。

使用 [ip-brief-template.md](references/ip-brief-template.md) 建立简短、可执行的内部 brief。仅在 `identity` 模式、用户要求系列复用，或参考图存在歧义时单独交付完整 brief。

### 3. 建立信息层级

`cover` 按以下顺序组织：

1. 主题标题；
2. IP 角色；
3. 必要的辅助模块；
4. 品牌署名。

`promo` 按以下顺序组织：

1. 主标题；
2. 产品、功能或活动主体；
3. IP 角色；
4. 一句卖点、副标题或行动语；
5. 品牌署名。

每张图只承担一个核心传播目标。标题是第一视觉层级，IP 用于识别和引导，不遮挡主题，也不让画面退化为角色设定图或功能说明书。

### 4. 选择构图

根据目标选择一种主要结构：

- **角色策展人**：IP 位于一侧，组织多个能力模块；
- **角色使用者**：IP 正在操作产品或体验功能；
- **角色引导者**：IP 指向标题、产品或核心成果；
- **品牌场景**：IP 融入统一世界观；
- **发布宣告**：大标题、产品主体和 IP 形成发布感；
- **结果展示**：IP 与最终成果同框；
- **场景故事**：用一个具体场景表达问题与解决方案；
- **系列主视觉**：一个母版派生多个统一规格版本。

详细适用场景和提示词结构见 [prompt-patterns.md](references/prompt-patterns.md)。

### 5. 选择生成与排字策略

- 标题短、文字少且允许轻微字形差异时，可以生成完整视觉；
- 中文较多、标题必须逐字准确或需要像素级排版时，先生成无字底图，再用可靠排版工具添加文字；
- 二维码、价格、日期、数据和产品界面必须来自可靠素材，并单独排版或合成；
- 多尺寸任务先完成一个高质量母版，再重排、扩图或裁切，不为每个平台重新生成角色。

调用图片生成或编辑能力时：

- 必须附带原始参考图；
- 明确要求保持同一角色身份，而不是生成相似风格的新角色；
- 包含目标尺寸、构图、安全区、主题层级、品牌色、材质、灯光和精确文本；
- 禁止额外文字、乱码、错误徽标、第三方 Logo 和角色漂移；
- 重要徽标无法稳定复现时优先简化，不生成近似乱码。

### 6. 派生多尺寸版本

不同尺寸保持：

- 同一 IP 身份、服装和核心配件；
- 同一主色、材质和光线；
- 同一标题语义和产品主体；
- 相近的角色姿势和镜头方向；
- 合理重排，而不是变成不同 Campaign。

小尺寸版本减少装饰和次要卖点，不把完整母版机械缩小成无法阅读的小字。

### 7. 验收

身份一致性：

- 第一眼是否仍是同一个 IP；
- 眼睛、耳朵、口鼻、轮廓和头身比例是否漂移；
- 主色、服装、徽标位置和材质是否一致；
- 手脚、尾巴、配件和发光结构是否合理；
- 是否混入参考图中的无关文字或布局。

传播和成图质量：

- 三秒内能否理解主题或核心卖点；
- 标题是否逐字准确、清晰且未被裁切；
- 缩小到实际展示宽度后是否仍可读；
- 产品主体和界面是否真实、完整；
- 是否出现虚假数据、错误 Logo、无来源承诺或多余文字；
- 多尺寸版本是否属于同一套视觉。

身份明显漂移、文字错误或主题不清时优先重新生成，不用后期解释掩盖。

### 8. 压缩与接入

使用 `agent-image-opt` 先检查和 dry-run，再写入发布文件。运行前先确认 CLI 可用：

```bash
agent-image-opt --json doctor
agent-image-opt --json inspect <image-or-directory> --recursive
agent-image-opt --json optimize <image-or-directory> --recursive --preset illustration --dry-run
agent-image-opt --json optimize <image-or-directory> --recursive --preset illustration --report image-opt-report.json
```

海报或多尺寸目录可使用 `poster` 预设；含细小文字时可改用 `ui` 预设并重新视觉复核。发布文件优先使用 WebP，同时保留高质量母版或源文件。

如果修改 README 或页面引用，验证文件存在、尺寸正确，并确认 Git diff 只包含目标资源和引用。

## 输出合同

根据模式返回必要内容：

- `identity`：IP brief、使用的参考图、已确认和未确认的特征、漂移检查结果；
- `cover`：封面文件、尺寸、标题、构图、身份检查、压缩结果和接入位置；
- `promo`：母版、平台版本、核心卖点、精确文案、身份与文字检查、压缩回执；
- 所有模式：明确未验证的产品界面、数据、外部素材或需要人工选择的风险。
