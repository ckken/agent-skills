# 构图与提示词模式

只选择一种主要视觉叙事，避免在同一画面中混合多个竞争结构。

## 角色策展人

适合工具合集、能力矩阵和 README 主视觉。IP 位于一侧或中心节点，以手势、视线或动线组织多个能力模块；标题保持第一视觉层级，模块不应像密集仪表盘。

## 角色使用者

适合产品、应用和具体功能。IP 正在操作产品主体，动作必须服务于真实使用场景；不要生成无法解释的按钮、连接结构或错误界面。

## 角色引导者

适合文章头图、发布页和成果说明。IP 的视线、手势或身体朝向引导受众阅读标题和核心成果，不遮挡文字。

## 品牌场景

适合活动、频道 Banner 和系列内容。IP 融入统一世界观，背景负责建立氛围，但不新增未经参考图确认的角色设定或品牌历史。

## 发布宣告

适合新功能、新版本和活动上线。使用大标题、明确产品主体和克制的发布氛围；每张图只表达一个发布信息。

## 结果展示

适合强调效率、质量或最终成果。IP 与成果同框，用视觉对比或聚焦表现收益；数据和效果承诺必须来自用户材料。

## 场景故事

适合把抽象功能转化为实际使用情境。只选择一个问题与解决结果，避免把完整流程或功能列表塞进主视觉。

## 系列主视觉

适合多平台和多尺寸 Campaign。先生成一个高质量母版，再重排、扩图或裁切；不要为每个平台重新创造角色身份和世界观。

## 提示词结构

按以下顺序组织生成提示词：

1. **身份**：引用原始参考图，并嵌入 IP brief 中的核心识别点；
2. **目标**：说明是封面还是宣传物料，以及受众和传播目标；
3. **画幅**：给出精确尺寸、主体位置、安全区和裁切要求；
4. **叙事**：指定一种构图模式及 IP 与产品、成果的互动；
5. **层级**：明确标题、产品主体、IP 和署名的优先级；
6. **风格**：指定品牌色、材质、灯光、背景和情绪；
7. **文字**：只包含经过确认的精确短文本；
8. **约束**：禁止角色漂移、额外文字、乱码、错误徽标和第三方 Logo。

长中文、二维码、价格、日期、数据和必须精确的界面不交给生成模型绘制，优先生成无字底图后单独排版或合成。
