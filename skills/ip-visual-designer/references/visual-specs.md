# 视觉物料尺寸参考

用户或平台提供的实时规格始终优先于本表。

| 场景 | 推荐尺寸 | 模式 | 说明 |
| --- | --- | --- | --- |
| GitHub README Hero | 1600 × 900 | cover | 16:9，标题和角色放在中部安全区 |
| 网站 Hero | 1920 × 1080 | cover | 预留响应式裁切空间 |
| 文章头图 | 1600 × 840 | cover | 约 1.91:1，适合多数内容平台 |
| 社交横幅 | 1500 × 500 | cover | 避免把脸和标题放在边缘 |
| 横版发布图 | 1600 × 900 | promo | 适合 GitHub、博客和发布页 |
| 公众号头图 | 900 × 383 | cover/promo | 宽幅文章入口或发布主视觉 |
| 视频封面 | 1280 × 720 | cover/promo | 小尺寸下标题仍需清晰 |
| 方形视觉 | 1080 × 1080 | cover/promo | 社交动态、频道入口和产品卡片 |
| 竖版宣传图 | 1080 × 1440 | promo | 移动端信息流 |
| Story 海报 | 1080 × 1920 | promo | 全屏竖版传播 |

## 安全区

- 标题与 IP 面部距离四边至少保留画面宽高的 5%；
- 横幅类视觉将核心内容控制在中央 70%；
- IP 面部、产品主体和标题不得同时贴近裁切边缘；
- 不把细小功能标签作为主要信息；
- 同一系列固定标题区、品牌署名和 IP 所在侧。

## 多尺寸派生

- 先完成横版或方形母版，再派生其他尺寸；
- 主标题尽量不超过两行；
- 竖版优先上下布局，横版优先左右布局；
- 小尺寸版本删除次要装饰和卖点，不机械缩小所有内容；
- 派生版本保持同一身份、服装、主色、光线和标题语义。
