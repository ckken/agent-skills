# GPT-5.6 提示词栈优化器

面向 GPT-5.6 系列审计、迁移并渐进优化项目的 Agent 提示词栈：覆盖 `AGENTS.md`、Agent 配置、Skills、工具或 MCP 描述、上下文与记忆规则、长会话状态和评测资产。

![GPT-5.6 提示词栈优化器推广图](./assets/promo.png)

## 解决什么问题

- 识别重复、冲突、失效或优先级不清的指令。
- 将稳定规则、项目不变量、按需能力、任务合同和阶段状态分层。
- 减少不必要的提示词、工具说明和上下文负担，同时保持质量与安全边界。
- 用相同任务集做优化前后对比，避免把“变短”误判为“变好”。

## 核心能力

| 能力 | 作用 |
| --- | --- |
| 12 维审计 | 从结果合同到可观测性定位提示词栈的结构性问题。 |
| 5 层架构 | 将规则与状态拆分为稳定底座、项目不变量、按需能力、任务合同和阶段状态。 |
| 4 种模式 | 支持只审计、出计划、实施、验收，并严格控制修改边界。 |
| 可验证输出 | 固定基线、复用任务集、区分实测与估算，并保留回滚点。 |

## 使用

在 Agent 任务中引用：

```text
使用 $optimize-prompt-stack 审计当前项目的 Agent 提示词栈，输出问题、证据与优化计划，不修改文件。
```

如需实施，明确授权范围与验收标准：

```text
使用 $optimize-prompt-stack 优化当前项目的提示词栈；仅修改 .codex/ 与 AGENTS.md，运行静态审计并给出前后对比。
```

## 包含内容

- [`SKILL.md`](./SKILL.md)：完整工作流、边界和输出合同。
- [`references/`](./references/)：审计量表、评测计划和优化模式。
- [`scripts/audit_prompt_stack.py`](./scripts/audit_prompt_stack.py)：静态盘点脚本。
- [`scripts/compare_audits.py`](./scripts/compare_audits.py)：前后审计结果对比脚本。

## 验证原则

只有在行为质量和安全边界未下降时，Token、延迟或调用数的减少才算优化。没有行为基线时，只报告静态变化，不对质量、成本或成功率作结论。
