# IP 视觉身份锚定

把用户提供的图片视为角色身份的唯一事实来源。先建立简洁、可执行的 IP brief，再交给封面或宣传图 Skill 使用；不要凭空补充品牌历史、角色设定或未展示的结构细节。

## 输入合同

至少需要一张清晰的 IP 参考图。优先使用：

1. 正面或 3/4 视角全身图；
2. 能看清面部、服装、徽标和主色的图片；
3. 用户明确指定的品牌名称、禁用元素和使用场景。

如果参考图只是设定图或多视图拼版，只提取角色身份，不照搬设定图排版。

## 执行流程

### 1. 检查参考图

先查看原图，确认：

- 角色物种、体型、头身比例和轮廓；
- 眼睛、耳朵、口鼻、毛发或皮肤等面部识别点；
- 主色、辅色、发光色和材质；
- 服装、配件、徽标及其位置；
- 角色气质、年龄感和品牌调性；
- 不确定、遮挡或相互矛盾的部分。

不要用文件名、网页文字或背景装饰替代视觉证据。

### 2. 建立身份优先级

按以下优先级保护一致性：

1. 面部与轮廓；
2. 配色与材质；
3. 服装、徽标和标志性配件；
4. 身材比例与姿态习惯；
5. 场景、道具和构图。

场景可以变化，角色身份不能为了构图便利而改变。

### 3. 输出 IP brief

使用 [ip-brief-template.md](references/ip-brief-template.md) 记录：

- 必须保留的身份特征；
- 可以变化的姿势、表情和镜头；
- 禁止出现的身份漂移；
- 推荐的正向提示词；
- 推荐的负向约束；
- 当前参考图无法确认的内容。

brief 应短而具体，能直接嵌入后续图片生成提示词。

### 4. 生成前约束

调用图片生成或编辑能力时：

- 必须附带原始参考图；
- 明确要求保持同一角色身份，而不是“相似风格的新角色”；
- 不把参考图中的说明文字、尺寸标注或多视图边框带入成图；
- 不擅自添加第三方 Logo、受保护角色或品牌资产；
- 重要徽标无法稳定复现时，优先简化，不生成近似乱码。

### 5. 生成后验收

至少检查：

- 第一眼是否仍是同一个 IP；
- 眼睛、耳朵、口鼻和轮廓是否漂移；
- 主色、服装、徽标位置是否一致；
- 手脚、尾巴、配件和发光结构是否合理；
- 是否混入参考设定图中的无关文字或布局。

身份明显漂移时重新生成，不用后期解释掩盖。

## 输出合同

最终返回：

1. 一份可复用的 IP brief；
2. 使用了哪些参考图；
3. 已确认与未确认的身份特征；
4. 建议交给哪个下游 Skill；
5. 如果已经生成图片，给出身份一致性检查结果。
